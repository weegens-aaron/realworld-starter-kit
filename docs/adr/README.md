# Architecture Decision Records (ADRs)

This directory holds Architecture Decision Records — short documents capturing a
significant decision, its context, and its consequences.

Each ADR is immutable once **Accepted**; to change a decision, add a new ADR that
**supersedes** the old one (and mark the old one accordingly).

## Format

Each ADR uses: **Status · Context · Decision · Rationale · Alternatives
Considered · Consequences**. Keep them short and decision-focused — link out to
code/beads for implementation detail.
